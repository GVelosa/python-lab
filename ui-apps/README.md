# UI Apps

This folder contains Python projects focused on building **Graphical User Interfaces (GUI)** and **Web Apps**, moving beyond the terminal to create interactive visual experiences.

These projects explore layout design, event handling, and reactive programming using modern Python frameworks.

## Project Development Patterns

While each application may vary in complexity—from single-page tools to multi-module platforms—all projects in this folder prioritize:

* **Reactive UI:** Built using state-driven frameworks like **Flet** or **Streamlit**.
* **Separation of Concerns:** Business logic is kept separate from UI layout whenever possible.
* **Self-Contained Projects:** Every directory is a standalone app with its own:
    * **README.md**: Detailing the specific objective, frameworks, and architecture.
    * **Requirements/Environment**: Clear instructions on dependencies (e.g., `requirements.txt`).
    * **Assets Management**: Proper handling of static resources (images, icons, etc.).

## Note

All projects in this folder are **GUI-based**, ranging from **single-purpose web pages** to **multi-functional platforms**. They are designed to be run as standalone applications or web tools, focusing on user interaction and visual feedback rather than terminal output. The architecture of each project is chosen to best fit its specific complexity and user needs.
