A programação orientada a objetos funciona com entidades que criamos a fim de replicar suas propriedade e métodos em instancias.

## Classes

Uma classe nada mais é do que um molde usado para criar objetos, vamos usar um canal no Youtube como exemplo:

1. Visitamos Class e depois dizemos o nome da nossa classe, exemplo Canal, assim criamos a classe
    
    ```python
    Class Canal
    ```
    
2. Depois de criarmos a classe criamos os atributos, que são as informações que compõem a classe
    
    ```python
    Class Canal
    nome
    descrição
    inscritos
    ```
    
3. Depois de definida a classe e seus atributos o que precisamos fazer é criar os Objetos

## Objetos

Para criar os Objetos utilizamos a classe como molde e a partir desse molde nosso objeto precisa respeitas os atributos dessa classe

1. A classe Canal possui 3 atributos: nome, descrição e inscritos, logo o canal que criamos precisa ter esses mesmos 3 atributos
    
    ```python
    Class Canal
    "Lan Code"
    "Canal de Programação"
    34.000,00
    ```
    

Com a teoria entendida vamos fazer uma descrição em código:

1. Criamos uma classe, é recomendado sempre fazer em CamelCase
    
    ```python
    class Canal:
    ```
    
2. Dentro da classe criamos uma espécie de função, mas dentro de classes elas são chamadas de métodos e esse método é onde definimos os atributos
    
    ```python
    class Canal:
        def __init__(self, nome, descricao, inscritos):
            self.nome = nome
            self.descricao = descricao
            self.inscritos = inscritos
    ```
    
    - Essa função init é o que chamamos de método construtor, ele é executado quando estamos instanciando.
    - O parâmetro self indica a si mesmo, no momento que instanciamos uma classe esse self passa a representar esse objeto que estamos criando, ele não representa a classe.
    - Todo método criado dentro de uma classe recebe o self
3. Com o método criado e os atributos definidos podemos criar o objeto
    
    ```python
    class Canal:
        def __init__(self, nome, descricao, inscritos):
            self.nome = nome
            self.descricao = descricao
            self.inscritos = inscritos
    
    canal_lancode = Canal("Lan Conde", "Canal de Programação", 34000)
    
    print(canal_lancode.descricao)
    ```
    
    - Nesse exemplo o self representa o canal_lancode.

## Métodos

No exemplo anterior usamos método __init__ esse método é o que chamados de construtor e é um método especial do python mas também podemos criar os nossos próprios métodos personalizados seguindo a mesma estrutura.

> Para o exemplo vamos usar a base do mesmo código anterior
> 
1. Fora do método init vamos criar outro mas ainda dentro da classe Canal

```python
class Canal:
    def __init__(self, nome, descricao, inscritos):
        self.nome = nome
        self.descricao = descricao
        self.inscritos = inscritos

    def inscrever(self, quantidade=1):
        self.inscritos +=quantidade

canal_lancode = Canal("Lan Conde", "Canal de Programação", 34000)

print(canal_lancode.descricao)
```

- O que esse novo método faz é pegar a quantidade atual de inscritos e somar com a quantidade, sendo a quantidade o que estamos enviando para ele que por padrão é 1
1. Agora vamos verificar o uso:
    
    ```python
    print(f"Quantidade e inscritos: {canal_lancode.inscritos}")
    canal_lancode.inscrever()
    print(f"Nova quantide de inscritos: {canal_lancode.inscritos}")
    ```
    
    - O que aconteceu aqui é que primeiro verificamos a quantidade de inscritos existente a partir do uso do primeiro método, o de criação, depois usamos o segundo método o de incremento de inscritos e então verificamos novamente a mesma informação a respeito dos inscritos. Nesse exemplo apenas 1 foi adicionado pois não passamos nenhum valor para o inscrever

## Herança

É quando criamos uma classe e usamos outra classe como referencia dela, basicamente criamos uma classe nova que herda tudo que outra classe tem, o diferencia é que podemos criar funcionalidades únicas para que apenas objetos criados com essa nova classe podem ter.

Vamos criar novas classes para esse exemplo fazer mais sentido

1. Nesse exemplo vamos criar uma classe de funcionários:
    
    ```python
    class Employee:
        def __init__(self, id, name, salary):
            self.id = id 
            self.name = name
            self.salary = salary
    
    employee_1 = Employee(12345, "Guilherme Sobrenome", 4000)
    
    print(employee_1.salary)
    ```
    
2. Agora vamos criar uma classe para os funcionários presentes na diretoria, eles ainda são funcionários mas om alguns atributos a mais por isso criamos uma nova classe que herda a classe Employee
    
    ```python
    class EmployeeDirector(Employee):
        def __init__(self, id, name, salary, team):
            super().__init__(id, name, salary)
            self.team = team
    ```
    
    - Primeiramente, mesmo que a classe nova herde tudo que a classe pai possua ainda precisamos criar um método construtor para ela pois sem ele ela não inicia direito.
    - Depois, esse super() representa a classe superior, a classe pai, quando usamos o super estamos chamando o método construtor da classe pai.
3. E agora podemos usar essa nova classe
    
    ```python
    class Employee:
        def __init__(self, id, name, salary):
            self.id = id 
            self.name = name
            self.salary = salary
    
    class EmployeeDirector(Employee):
        def __init__(self, id, name, salary, team):
            super().__init__(id, name, salary)
            self.team = team
    
    employee_1 = Employee(12345, "Guilherme Sobrenome", 4000)
    employee_2 = Employee(52134, "Ruan Sobrenome", 4500)
    employee_3 = Employee(54321, "Lanilsson Sobrenome", 2300)
    
    director_1 = EmployeeDirector(
        11111, 
        "Fernando Sobrenome", 
        40000, 
        [employee_1.id, employee_2.id, employee_3.id]
        )
    
    print(employee_1.salary)
    print(director_1.team)
    ```

## Encapsulamento

Existem propriedade que não queremos que sejam alteradas com facilidade e vamos querer fazer um tratamento antes de adiciona, e parra isso usamos o conceito de encapsulamento, isso é, pegar uma propriedade, por exemplo, o ‘team’, criar métodos em cima dela para deixarmos os processos mais seguros.

Seguindo o exemplo anterior, queremos verificar se uma mesma pessoa não foi adicionada duas vezes na mesma equipe.

1. Primeiro vamos mudar o nome do team para _team, esse _ é uma convenção da comunidade para indicar que a propriedade não deve ser mexida diretamente, ela só pode ser alterada dentro d apropria classe
    
    ```python
    class EmployeeDirector(Employee):
        def __init__(self, id, name, salary):
            super().__init__(id, name, salary)
            self._team = []
            
        @property
        def team(self):
            return self._team
    ```
    
    - O @property serve no python para indicar que o team é uma propriedade e não precisamos ficar usando o () na hora de chama-lo
2. E agora vamos criar a função de adicionar um membro na equipe que checa se o membro existe ou não
    
    ```python
    class EmployeeDirector(Employee):
        def __init__(self, id, name, salary):
            super().__init__(id, name, salary)
            self._team = []
    
        @property
        def team(self):
            return self._team
        
        def add_team(self, member):
            if member not in self._team:
                self._team.append(member)
            else:
                print("This Member Alredy Exists")
    ```
    
3. Agora podemos usar a classe com a adição de membros encapsulada:
    
    ```python
    class Employee:
        def __init__(self, id, name, salary):
            self.id = id 
            self.name = name
            self.salary = salary
    
    class EmployeeDirector(Employee):
        def __init__(self, id, name, salary):
            super().__init__(id, name, salary)
            self._team = []
    
        @property
        def team(self):
            return self._team
        
        def add_team(self, member):
            if member not in self._team:
                self._team.append(member)
            else:
                print("This Member Alredy Exists")
    
    employee_1 = Employee(12345, "Guilherme Sobrenome", 4000)
    employee_2 = Employee(52134, "Ruan Sobrenome", 4500)
    employee_3 = Employee(54321, "Lanilsson Sobrenome", 2300)
    
    director_1 = EmployeeDirector(
        11111, 
        "Fernando Sobrenome", 
        40000
        )
    
    print(f"Current team members: {director_1.team}")
    for i in (employee_3.id, employee_2.id, employee_1.id):
        director_1.add_team(i)
    print(f"Current team members: {director_1.team}")
    director_1.add_team(employee_3.id)
    print(f"Current team members: {director_1.team}")
    ```

class Channel:
    def __init__(self, name, description, subscribers):
        self.name = name
        self.description = description
        self.subscribers = subscribers
        self.videos = []
        self.playlists: list[Playlist] = []
    
    def subscribe(self, quantity=1):
        self.subscribers += quantity
    
    def post(self, video):
        if video in self.videos:
            print("This video already exists!")
            return
        self.videos.append(video)
    
    def playlist_info(self):
        for playlist in self.playlists:
            print(playlist.name)
            playlist.video_info()
    
    def add_playlist(self, playlist):
        if playlist not in self.playlists:
            self.playlists.append(playlist)
        else:
            print("This playlist has already been added!")
    
    def remove_playlist(self, playlist):
        if playlist in self.playlists:
            self.playlists.remove(playlist)
        else:
            print("This playlist does not exist.")
